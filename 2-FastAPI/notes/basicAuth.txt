# ============================================================
# BASIC AUTHENTICATION — FASTAPI
# ============================================================
#
# This file explains how HTTP Basic Authentication works in FastAPI.
# It is designed for learning how authentication, dependency injection,
# and security handling work in backend APIs.
#
# ============================================================
# CORE CONCEPTS & MODULES
# ============================================================
#
# 1. FastAPI
#    - Creates the API application instance.
#    - Main server object that handles requests.
#
# 2. HTTPBasic
#    - Enables HTTP Basic Authentication.
#    - Reads username and password from the Authorization header.
#
# 3. Depends
#    - FastAPI dependency injection system.
#    - Allows reuse of authentication logic across routes.
#
# 4. HTTPBasicCredentials
#    - Contains username and password from the request.
#    - Automatically parsed by FastAPI.
#
# 5. HTTPException
#    - Used to return HTTP errors like 401 Unauthorized.
#
# 6. status.HTTP_401_UNAUTHORIZED
#    - Standard HTTP response code for failed authentication.
#
# 7. secrets.compare_digest()
#    - Secure way to compare strings.
#    - Prevents timing attacks compared to using "==".
#
# ============================================================
# AUTHENTICATION FUNCTIONS
# ============================================================
#
# verify_password()
# - Compares input password with stored password securely.
#
# get_current_username()
# - Core authentication function.
# - Steps:
#     1. Extract username and password
#     2. Check if username exists
#     3. Validate password
#     4. Return username if valid
# - Raises 401 error if authentication fails.
#
# ============================================================
# PROTECTED ROUTES
# ============================================================
#
# Any route using:
#     Depends(get_current_username)
#
# means:
# - User must provide valid credentials
# - Otherwise access is denied
#
# Example protected endpoints:
# - /users/me
# - /secure-data
#
# ============================================================
# AUTHENTICATION FLOW
# ============================================================
#
# Client Request
#       ↓
# Authorization Header (Basic Auth)
#       ↓
# HTTPBasic extracts credentials
#       ↓
# get_current_username validates:
#       - username exists?
#       - password correct?
#       ↓
# Access granted OR 401 Unauthorized
#
# ============================================================
# SECURITY NOTES
# ============================================================
#
# - Basic Auth sends credentials on every request.
# - Base64 encoding is NOT encryption.
# - Always use HTTPS in production.
# - Never store plain-text passwords in real systems.
# - Use hashed passwords (bcrypt / passlib).
# - Use databases instead of in-memory dictionaries.
