# ============================================================
# FASTAPI PYDANTIC SCHEMAS LESSON
# ============================================================
#
# This file demonstrates how to use:
# - Pydantic models
# - Data validation
# - Enums
# - Optional fields
#
# These schemas define the structure and validation
# rules for API request and response data.
#
# ------------------------------------------------------------
# MODULES AND FUNCTIONS USED
# ------------------------------------------------------------
#
# 1. BaseModel
#    - Main Pydantic model class.
#    - Used for request and response validation.
#
# 2. Field()
#    - Adds validation rules to model fields.
#    - Example:
#        • min_length
#        • max_length
#        • default values
#
# 3. Enum
#    - Restricts values to predefined options.
#    - Helps maintain consistent API data.
#
# 4. Optional
#    - Allows fields to be optional or nullable.
#    - Commonly used for update operations.
#
# ------------------------------------------------------------
# ENUM CLASSES EXPLAINED
# ------------------------------------------------------------
#
# IssuesStatus
# - Defines allowed issue statuses:
#     • open
#     • in_progress
#     • closed
#
# IssuePriority
# - Defines allowed priority levels:
#     • low
#     • medium
#     • high
#
# Enums help:
# - prevent invalid values
# - improve API consistency
# - simplify frontend/backend communication
#
# ------------------------------------------------------------
# PYDANTIC MODELS EXPLAINED
# ------------------------------------------------------------
#
# IssueCreate
# - Used when creating new issues.
# - Includes validation rules:
#     • title length
#     • description length
# - Automatically sets default priority to "medium".
#
# IssueUpdate
# - Used for updating existing issues.
# - All fields are optional.
# - Allows partial updates without requiring all fields.
#
# IssueOut
# - Defines the API response structure.
# - Controls how issue data is returned to clients.
#
# ------------------------------------------------------------
# WHY PYDANTIC IS IMPORTANT
# ------------------------------------------------------------
#
# Benefits:
# - Automatic validation
# - Cleaner backend logic
# - Strong data consistency
# - Better API documentation
# - Type safety
#
# FastAPI automatically:
# - validates incoming JSON
# - generates Swagger documentation
# - returns validation errors
#
